### 画像が表示されない問題 — 診断を間違えた

置いてあるのにプレースホルダーが出た。最初「Propshaft は最新 mtime で判定するので、
後から別ファイルを編集すると検知されなくなる」と説明したが、**これは誤りだった**。

`ActiveSupport::FileUpdateChecker#updated?` を読むと、

```ruby
if @last_watched.size != current_watched.size
  true          # ← ファイル数が変われば mtime に関係なく検知される
```

単純な追加は必ず検知される。実際、再現しようとしても普通に表示されてしまった。
**再現手順が2回とも間違っていた**（`touch` した別ファイルが再スキャンを
引き起こしていただけで、修正の効果を見ていなかった）。

正しい条件は **「ファイル数が変わらず、かつ mtime も古い」** ときだけ。
1枚消して1枚足す、差し替える、といった操作で起きる。これは再現できた。
見つからなかったときだけ一覧を作り直して再試行するようにした（開発環境限定）。

なお、実際に `beginning.jpg` でこの条件が成立していたかは**確認できていない**。
症状と対処は一致しているが、引き金が同じだった確証はない。

### つまずいた点

- **モーダルが閉じなかった** … `.modal { display: flex }` が、閉じた `<dialog>` に
  ブラウザが当てる `display: none` を打ち消していた。`.modal[open]` にして解決。
  最初の修正は的外れで、ブラウザキャッシュのせいにしてしまった
- **モーダルが勝手に開いた** … Stimulus の Boolean は**属性があるだけで true**。
  `data-modal-open-value="false"` でも開く。エラー時だけ属性を出すようにした
- **ボタンが画面外に押し出された** … `margin-top: auto` と `position: sticky` は
  相性が悪い。スクロール領域と固定フッターに分けた
- **一覧へもどるが戻らなかった** … `:back`（`history.back()`）は POST 後の履歴では
  意図した場所に戻らない。`index_path` を直接指定した
