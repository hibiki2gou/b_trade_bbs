# B-Trade BBS — 企画とこれまでの流れ

## ひとことで言うと

Bリーグカード（B.LEAGUE 公式のデジタルカードコレクション）をやっている人どうしが、
カードのトレード相手を見つけるための掲示板。
ダブったカードを無駄にせず、欲しい人へ橋渡しする「カードの物々交換所」。

## なぜ作るのか

Bリーグカードは基本ガチャで集めるが、狙ったカードが出ないことが多い。
そこで用意されているのがユーザー同士のトレード。
でも「誰が何を欲しがっていて、誰が何を譲れるのか」を知る場所がないと、相手が見つからない。
そこを掲示板でつなぐのが、このアプリの出発点。

## 想定ユーザー

- Bリーグカードをプレイしていて、欲しいカードがガチャで出なくて困っている人
- ダブったカード・手放してもいいカードを活用したい人
- 気軽にトレード相手を探したい、Bリーグ／バスケ好きなファン

## 解決したい困りごと

- 欲しいカードを持っている相手が見つからない
- 自分のダブりカードを欲しがっている人に出会えない
- トレードの募集や交渉を書き込める、共通の場所がない

## これまでの流れ（2026-07-16〜17）

このプロジェクトは、設計を会話で詰めてから実装に入る形で進めてきた。
主要な論点と、そこで下した判断は [roadmap.md](roadmap.md) の「決定事項」にまとめてある。
ここでは大きな流れだけ記す。

### 1. 環境構築（完了）

- Rails 8.1.3 / Ruby 3.4.10 / SQLite / Docker で開発環境を構築
- ホスト側に Ruby / Rails を入れず、すべて Docker の中で動かす
- `docker compose up` → http://localhost:3001

### 2. 設計の作り込み（完了）

会話を通じて、当初の素朴な企画から以下の点が洗練された。

- **欲しい／譲れるを1つの投稿に統合**した（カテゴリで分けない）
- **弾（カードセット）をスレッドの単位**にし、運営が事前に立てる（過疎対策）
- **選手・弾・カードを正規化**し、情報の重複を排除した
- **トレード成立ボタン**で募集を非表示にし、成立記録はレート表として残す
- **アカウントIDは公開しない**方針を決めた（ログイン必須・本人のみ表示）

### 3. モデルとマスタデータ（完了）

- Board / Topic / Player / Card / Post の5モデルを実装
- `db/seeds/card_sets.yml` から検証つき・冪等にデータ投入
- `rails console` で全関連・バリデーションの動作を確認済み

### 4. これから

画面（3画面＋投稿フォーム）の実装に入る。詳細は [roadmap.md](roadmap.md)。
