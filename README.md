# TRADE COURT

**Bリーグカードのトレード相手を見つけるためのWebアプリ。**

ダブったカードを無駄にせず、欲しい人へ橋渡しする「カードの物々交換所」。
Bリーグカードは狙ったカードがガチャで出ないことも多く、ユーザー同士のトレードが用意されている。
けれど「誰が何を欲しがっていて、誰が何を譲れるのか」を知る場所がないと相手は見つからない。
そこをつなぐのがこのアプリ。

## できること

- **弾（カードセット）ごと**の掲示板で、その弾のカードのトレード相手を探す
- **クラブチームごと**の掲示板でも探せる（弾をまたいで、そのチームのカードを欲しがっている人が見つかる）
- 募集には**欲しいカード**と**出せるカード**を、どちらも選手名で検索して複数枚まで指定できる
- 収録カード一覧で、その弾／チームにどんなカードがあるかを把握できる
- トレードが決まったら**成立ボタン**で募集を閉じられる（成立した組み合わせは記録として残り、レートの参考になる）

### 画面の流れ

```
トップ（弾一覧）  ⇄  チーム一覧      ← ボタンで切り替え
   ↓ 弾/チームを選ぶ
掲示板（募集一覧 ＋ 収録カード一覧 ＋ 募集フォーム）
```

**投稿は1つ、入口は複数。** 1件の募集は、欲しいカードが属する弾の掲示板と、その選手のチームの掲示板の
両方に自動的に現れる。二重に投稿する必要はない。

## 技術構成

| | |
|---|---|
| Rails | 8.1.3 |
| Ruby | 3.4.10 |
| DB | SQLite 3 |
| フロント | Hotwire（Stimulus）／CSS は自前 |
| 実行環境 | Docker / Docker Compose |

Ruby も Rails も Docker の中にだけ入っている。ホスト側に入れる必要はない。

## 使い方

初回だけイメージのビルドが要る。

```
docker compose up --build
```

2回目以降はこれだけ。

```
docker compose up
```

起動したらブラウザで **http://localhost:3001** を開く。止めるときは `docker compose down`。

> ポートが3000ではなく3001なのは、別プロジェクトが3000番を使っていてぶつかるため。
> 3000番が空いたら `compose.yaml` の `ports` を `"3000:3000"` に戻してよい。

## よく使うコマンド

コンテナの中で実行するので、頭に `docker compose exec web` を付ける。

```
docker compose exec web bin/rails console      # コンソール
docker compose exec web bin/rails db:seed      # マスタデータの投入
docker compose exec web bin/rails test         # テスト
docker compose exec web bin/rubocop            # コードスタイルの検査
docker compose exec web bash                   # シェルに入る
```

コードはホストからマウントしているので、**エディタで編集すればそのまま反映される**。
build が要るのは `Dockerfile.dev` を変えたときだけ。`Gemfile` に gem を足したときは起動時に自動で `bundle install` が走る。

## データの構造

```
Team（チーム）──< Player（選手）──< Card（カード）>── Topic（弾）
                                      ↑        ↑
                        Post（募集）──┴────────┘
                          ├─ wanted_cards  … 欲しいカード（複数）
                          └─ offered_cards … 出せるカード（複数）
```

- **Topic（弾）** … カードセット。掲示板の単位。例: OCEAN
- **Team（チーム）** … クラブ。こちらも掲示板の入口になる
- **Player（選手）** … 背番号・ポジション・所属チームを持つ
- **Card（カード）** … 選手 × 弾 の交差点。レア度（★1〜5）を持つ
- **Post（募集）** … 欲しい／出せるカードを複数指定できる書き込み

募集がどの掲示板に出るかは、**欲しいカードから導かれる**（弾とチームの両方）。

## マスタデータの追加

弾・チーム・選手・カードは [db/seeds/card_sets.yml](db/seeds/card_sets.yml) にまとめてある。
編集して `bin/rails db:seed` を実行すれば反映される（何度実行しても同じ結果になる）。

- **新しい弾** … `sets` の末尾に追記する。画面では自動的に一番上（最新）に並ぶ
- **カードの収録内容** … `lineups` に弾ごとのラインナップを追記する
- 書き間違い（選手名の表記ゆれ、レア度の範囲外など）は seed が検出して中断する

## 画像

弾とチームの画像は、slug と同じ名前で置くと自動的に表示される（無ければ名前のタイルになる）。

```
app/assets/images/sets/<slug>.jpg     … 弾のバナー
app/assets/images/teams/<slug>.png    … チームのロゴ
```

ファイル名の対応表は [docs/image_filenames.txt](docs/image_filenames.txt) にある。
なお**これらの画像は公式素材のため git 管理外**にしてある（`.gitignore` 済み）。
